Fonctionnement du site

Introduction

Le site “Bienvenuealamaison” (BALM) est une plateforme de location et d’échange de biens immobiliers mettant en relation directement les particuliers. Ainsi, si le site s’apparente à un AirBNB, il n’y a aucune gestion d’échange d’argent et les utilisateurs sont directement mis en relation, à la manière du site Leboncoin. Notre projet se base sur un ancien site, aujourd'hui abandonné, portant le même nom.

         Page d'accueil de l’ancien site                        Page d'accueil du nouveau site

Il propose donc un service complet accompagnant chaque utilisateur jusqu’au bout de la démarche, qu’il recherche un bien ou qu’il souhaite en mettre un à disposition.

Présentation structure du code

Choix de la conception

Pour construire notre projet, nous avons choisi d’utiliser Express qui est aujourd’hui le constructeur le plus utilisé.

Pour le front-end: nous avons utilisé le langage pug, alternative épuré et performante de HTML qui se convertit automatiquement. Il est accompagné de CSS et de Javascript.

Pour le back-end, nous avons choisi un framework javascript car il y a une énorme communauté et cela nous fait un langage en moins à manipuler.

Pour le système de stockage de données, nous avons opté pour une base de donnée SQL car c’est une solution répandue et efficace.

Arborescence

L'arborescence de notre code est donc celui proposé par défaut lors de la création d’un projet via Express.

bin contient le fichier permettant le lancement du serveur, ouvrant la connection et l’écoute sur un port défini.

models correspond à l’ensemble des fichiers effectuant les requêtes SQL. Leur rôle est donc d’accéder à la base de données et de retourner la ou les informations souhaitées.

public contient l’ensemble des fichiers bruts ( telle que les images ), ainsi que les fichiers nécessaires aux pages HTML ( fonctions Javascripts, fichiers CSS )

routes défini quant à lui les communications entre le client et le serveur. Ainsi, les fichiers présents dans ce dossier contiennent les instructions à effectuer pour chacune des requêtes clients.

sql contient tout ce qui relève de la base de données.

views est le dossier stockant tous les fichiers .pug.

Présentation des différentes pages

La barre supérieure

La barre supérieur est présente sur toutes les pages, et elle permet un accès rapide à la page d'accueil ( via le bouton accueil ou BALM ), un accès à la page de connexion (“login”), ainsi que le profil utilisateur ( “user”) si ce dernier est connecté.

La barre inférieure

Une barre inférieure est présente sur toutes les pages, sur celle-ci se trouve un bouton d’aide. Lorsque l’on clique dessus, cela va ouvrir une fenêtre proposant un petit tutoriel sur l’utilisation du site web.

La page d'accueil

La page d'accueil est composée d’une barre de recherche principale, permettant aux utilisateurs de rechercher une destination ( ville, departement ou region ). Lorsque l’utilisateur commence à rentrer une destination, les suggestions vont lui être immédiatement proposé afin de l’aider si besoin. Cela est fait à l’aide d’une requête envoyée au serveur, qui compare ce que l’utilisateur a entré avec la base de donnée ( les villes, département et régions y étant tous stocké ) , et qui renvoie les suggestions les plus adéquates.

Cette dernière renvoie donc vers la page “search”, présentant les biens correspondant à la recherche.

 

Il est possible d’affiner la recherche avec le bouton “options avancées”, ouvrant sur un ensemble de paramètres permettant de mieux trier les résultats de la recherche. Ces paramètres sont date de départ de d’arrivée, nombre de personnes et des options matériels comme la présence d’un jardin ou d’une piscine. Il est aussi possible de choisir le type de services souhaité : échange, location ou hébergement.

Page de présentation d’un bien

Chaque bien présent sur le site possède un page détaillé le décrivant.

On y retrouve :

  • des images du bien pour permettre au potentiel client de s’y imaginer,
  • un titre choisi par le propriétaire,
  • le lieu géographique du bien,
  • la moyenne des notes obtenue par les visiteurs précédents,
  • une description détaillée du bien fourni par le propriétaire,
  • une section avec tous les commentaires et les notes détaillé donné par chaque client précédent
  • un encadré pour faire une demande de réservation du bien pour une période et un nombre de jours
  • le prix par nuit ou le signalement d’un service d'hébergement ou d’échange

Cette page s’affiche suite à la sélection d’un bien après avoir effectué une recherche. Lorsque l’on clique sur un bien, son numéro d’identification unique est transmis à la page d’affichage du bien.

Lorsque l’utilisateur clique sur un bien, quatre requêtes SQL sont envoyées à la base de données. Celles-ci nous retournent toutes les images du bien depuis la table image, toutes les informations de description du bien depuis la table Offer, les périodes de disponibilité du bien depuis la table Availability et les commentaires laissés par les utilisateurs à propos de ce bien depuis la table Comment.

Aussi sur cette page il est possible de faire une demande de réservation d’un bien auprès de son propriétaire. Pour faire une demande, il faut que l’utilisateur soit connecté (sinon il est renvoyé sur la page de connexion), qu’il ait rentré des dates valides et qu’il rentre un nombre de personnes voyageant avec lui. S’il saisit les informations correctement, une nouvelle ligne sera alors ajoutée à la table Reservation contenant l’id de l’utilisateur, l’id du bien, les dates rentrées, le nombre de personnes voyageant, ainsi qu’un flag ayant pour valeur 0, ce qui traduit que la demande a été prise en compte et que l’on attend une validation de la part du propriétaire.

Page de confirmation de demande de réservation

Lorsqu’une demande de réservation a été envoyée et enregistrée dans la base de données, l’utilisateur voit s’afficher sur son écran une confirmation de demande de réservation contenant le récapitulatif de la demande : les dates de départ et d’arrivée ainsi que le nombre de personnes voyageant.

Page de connexion et de création d’un nouveau compte

Pour pouvoir utiliser les services de ce site web, l’utilisateur est obligé d’avoir ou de se créer un compte. Pour créer un compte, l’utilisateur doit renseigner son nom, son prénom, une adresse mail et un mot de passe. Son adresse mail sera son identifiant de connexion. Son mot de passe et son identifiant lui seront demandés pour chaque action qu’il veut faire sur le site web, à l’exception du lancement d’une recherche et de l’accès à la fiche descriptive d’un bien immobilier. Toutes les informations citées ci-dessus et concernant un utilisateur, sont conservées dans la table User de la base de données.

Pages des recherches

En ce qui concerne la page de recherche, celle-ci est accessible lorsque nous effectuons une recherche à partir de la page de garde. Les critères des recherches sont les suivants :

  • La localisation. Il est possible de renseigner une ville, un département ou une région afin d’élargir la localisation si certain utilisateurs le souhaite.
  • La date de départ ou d’arrivé de l’utilisateur.
  • La date de fin.
  • Le nombre de personne minimum que le client souhaite en nombre de capacité
  • Des critères avancés afin de viser plus précisément des biens pour les utilisateurs les plus exigeants. On peut sélectionner si l’on souhaite avoir une piscine, un jardin ou que le biens soit situé en centre-ville.

La page de recherche récupère tous les critères saisis par l’utilisateur et effectue une requête auprès de la base de données afin d’en ressortir toutes les annonces qui correspondes à la demande. Pour cela nous avons fait en sorte que la requête auprès de la base de données corresponde exactement aux critères saisis par les utilisateurs.

La page est ordonnée de la manière suivante : En haut de la page se situe un petit récapitulatif de la recherche à savoir la localisation saisis, les dates et les options avancées, ainsi que le nombre total annonces trouvées.

Nous avons décidé de faire une design très épuré et non trop chargé par les annonces c’est pour cela que nous avons décidé d’afficher que 6 offres par pages. Pour cela nous avons dû créer un système de pagination dans le cas d’un nombre d’annonces retournées supérieur à la capacité de la page.

Chaque annonce est affichée dans un petit emplacement mettant en avant la photo principale. En dessous de celle-ci on peut retrouver les informations importantes la concernant à savoir : son titre, son prix, sa capacité humaine mais aussi une note sur 5 étoiles regroupant tous les commentaires et les notes laissés par les utilisateurs ayant déjà réservé ce bien.

Il suffit de cliquer sur l’annonce intéressée pour accéder à la page personnelle de ce bien.

Un bouton permet d’effectuer une nouvelle recherche est placé en bas de la page.

Page utilisateur

Chaque utilisateur possède un espace personnel regroupant toutes les informations le concernant. La page se décompose sous un ensemble d’onglet : les informations personnelles, les offres qu’il a publié, ses demande de réservations, les commentaire qu’il a laissé  et un onglet pour accepter des demandes de réservations de la part des utilisateurs. En accédant à ses annonces, l’utilisateur va aussi avoir l’opportunité de les modifier.

Les différents onglets ont été réalisés grâce à la technologie AJAX qui permet de réaliser des requêtes HTTP de manière asynchrone sans recharger une page entière. Les onglets, du côté serveur, sont assez simples: une route basique, la récupération des paramètres (l’id de l’utilisateur), une requête SQL et la génération d’un document.

La page ‘information personnelles’ permet d’afficher ses informations (prénom, nom et email). Elle pourra aussi servir à modifier son mot de passe. ‘Mes offres’ permet d’afficher la liste des offres que l’utilisateur a déposé sur le site. Cette page permet aussi de modifier/supprimer chaque offre. ‘Mes commentaires’ permet de voir les commentaires que les gens ont laissé sur ses offres. ‘Mes réservation’ permet de voir les réservations en attente sur ses biens et ‘Mes demandes’ permet de voir les demandes réservation que l’utilisateur à fait.

Onglet commentaires

Déposer une annonce

Une fois que l’utilisateur est connecté, il peut déposer une nouvelle annonce pour son bien. En effet, sur la barre de recherche en haut de son écran apparaît un nouveau bouton pour déposer une annonce. Il est alors redirigé vers une page de formulaire à remplir avec toutes les informations concernant son bien. Il peut ajouter à son annonce 3 images servant à illustrer son bien. Les données complétées seront alors insérées dans la base de données dans les tables offres, périodes de disponibilités et images.

 

Modification d’une annonce

La page de modification d’une annonce est similaire à la page précédente. Cependant, les cases sont déjà préremplies avec les informations actuelles concernant l’offre, et certaines informations telles que le titre de l’offre, ou l'adresse, ne peuvent pas être modifiées.

Présentation de la base de donnée

Boolean Hebergement

Boolean Echange

En ce qui concerne la base de données, nous avons décidé de travailler avec un serveur MySQL pour plusieurs raisons. Dans un premier temps MySQL est un système de gestion de base de données relationnelles, ce qui va nous permettre de mettre en relation toutes nos tables. Mais aussi du fait que MySQL est le langage le plus utilisé car celui offre de grandes personnes et notamment sa rapidité d'exécution en ce qui concerne la complexité des requêtes.

Notre base de données est décomposée est 8 tables, dont une principale qui est la table “Offre”. C’est autour de cette table que toutes les autres tables seront en lien. Cette table contient toutes les informations nécessaire concernant une annonce telles que : l’utilisateur qui a déposé l’offre, ses informations géographiques et ses caractéristiques. Chaque offre est référencée de manière unique par un id qui lui est attribué.

A la table des offres, nous avons rajouté par la suite deux valeurs boolean, hébergement et échange. Celles-ci permettent aux utilisateurs de proposer des services d'hébergement gratuit ou d’échange de maison.

Nous avons optimisé notre base de données afin d’éviter les doublons et la rendre plus performante. A partir du nom de la ville saisit dans une annonce, il en est automatiquement déduit le département ainsi que la région dont l’offre fait fonction.

Nous avons décidé de mettre dans des tables à part les disponibilités, les commentaires, les réservations et les images des offres, afin d’avoir une meilleur gestion des données mais aussi du fait qu'une annonce peut avoir plusieurs images, réservations , etc …

En ce qui concerne les utilisateurs nous avons créé une table les concernants en relation avec la table offre. Cette table contient toutes les informations concernants les utilisateurs inscrits sur notre site web. Ces informations sont leur nom, prénom, email et leur mot de passe. Ce dernier est haché lors de la création de l’utilisateur afin de ne pas sauvegarder un mot de passe en clair.

Conclusion :

Nous avons atteint nos objectifs initials, c’est-à-dire de remettre en état et au goût du jour un site internet datant de 2006. Notre site est fonctionnel, facile d’utilisation et il garantit une sécurité des données grâce au cryptage des informations sensibles et à l’utilisation du protocole HTTPS.
